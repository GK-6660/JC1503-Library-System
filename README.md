# 图书管理系统 (Library Management System) - 操作指南

本项目是一个基于 Python 和自定义数据结构实现的命令行图书管理系统。系统旨在模拟真实的图书馆借阅流程，包含用户管理、资源（书籍/杂志）管理、借阅与归还、等待队列以及操作撤销等核心功能。

---

## 🛠️ 1. 运行环境与启动方式

- **环境要求**: Python 3.6 或更高版本。
- **启动方式**: 打开终端，导航到项目根目录，运行以下命令启动系统主程序：
  ```bash
  python src/main.py
  # 或在 macOS/Linux 上使用：
  python3 src/main.py
  ```

---

## 🚀 2. 初次运行与自动初始化数据

当您**第一次**运行系统时（即 `data/library_data.json` 文件不存在时），系统会自动生成一个大型初始数据集，以便您直接进行测试。
初始数据集包含：
- **30 名用户**: ID 格式为 `U001` 到 `U030` (例如: `U001`, `User_Name_1`)。
- **40 本书籍**: 标题格式为 `Book_Title_1` 到 `Book_Title_40`。
- **20 本杂志**: 标题格式为 `Magazine_Title_1` 到 `Magazine_Title_20`。
- **初始借阅记录**:
  - 用户 `U001` 已借阅书籍 `Book_Title_1`。
  - 用户 `U002` 已借阅杂志 `Magazine_Title_1`。

系统启动后，会显示主菜单界面。

## 📖 3. 功能菜单详解与逻辑说明

主菜单包含以下选项，通过输入对应的数字即可执行：

### `1. Add user` (添加用户)
- **操作逻辑**: 在哈希表 (Hash Table) 中注册一个新用户。
- **输入要求**:
  - `ID`: 用户的唯一标识符 (例如: `U099`)。注意：ID 区分大小写。
  - `Name`: 用户的姓名。

### `2. Add book` (添加书籍/杂志)
- **操作逻辑**: 将新资源插入到二叉搜索树 (BST) 中，使用标题作为查找关键字。
- **输入要求**:
  - 选择类型: 输入 `1` 添加书籍 (Book)，输入 `2` 添加杂志 (Mag)。
  - `ID`: 资源的唯一编号。
  - `Title`: 资源的标题。**注意：这是后续借阅/归还时查找的唯一关键字，需精确匹配**。
  - `Total`: 总副本数量 (输入整数)。
  - 根据类型补充信息：书籍需输入 `Author` (作者) 和 `ISBN`；杂志需输入 `Issue` (期号)。

### `3. Borrow book` (借阅资源)
- **操作逻辑**:
  1. 通过哈希表查找用户。
  2. 通过 BST 查找资源（书籍/杂志）。
  3. 如果资源有库存，库存减 1，并将该资源标题加入用户的借阅链表 (Doubly Linked List) 中。
  4. **如果资源无库存 (Out of stock)**: 系统会询问是否加入等待队列 (Waitlist)。输入 `y` 将当前用户 ID 加入该资源的等待队列 (Queue) 中。
- **输入要求**:
  - `User ID`: 借阅者的 ID (如 `U001`)。
  - `Title`: 想要借阅的资源标题 (如 `Book_Title_5`)。

### `4. Return book` (归还资源)
- **操作逻辑**:
  1. 验证用户是否真的借阅了该资源。
  2. 如果验证通过，将资源从用户的借阅链表中移除。
  3. **等待队列机制触发**: 资源库存加 1 后，系统会立即检查该资源的等待队列 (Queue)。如果队列不为空，系统会自动将该资源借给队列中的下一位用户，并打印通知信息。
- **输入要求**:
  - `User ID`: 归还者的 ID。
  - `Title`: 归还的资源标题。

### `5. Undo` (撤销最后一次操作)
- **操作逻辑**: 
  - 系统使用一个栈 (Stack) 来记录所有的增、删、借、还操作历史。
  - 选择此功能将弹出 (pop) 栈顶的最后一次操作记录，并执行反向逻辑（例如：刚借的书会被自动归还，刚添加的用户会被移除）。
  - **注意**: 如果撤销的是“归还”操作，系统会自动让原用户重新借出该书；如果撤销的是“借阅”操作且触发了等待队列，系统会尝试妥善处理后续用户的借阅状态。

### `6. Show History` (查看操作历史)
- **操作逻辑**: 打印历史记录栈 (Stack) 中的内容，展示系统运行期间发生的所有操作，**最新的操作显示在最前面**。这对于追踪系统的操作轨迹非常有用。

### `0. Exit` (退出系统)
- **操作逻辑**: 
  - 结束主循环。
  - 触发 `Storage` 模块的保存机制，调用所有对象和数据结构的 `to_dict()` 与 `to_list()` 方法。
  - 将当前系统状态（包括所有用户、资源、库存、等待队列和借阅记录）序列化并保存到 `data/library_data.json` 文件中。
  - 下次启动系统时，将自动从该文件恢复状态。

## ⚠️ 4. 注意事项与排错 (Troubleshooting)

1. **输入精确性**: 在执行借阅 (Borrow) 和归还 (Return) 操作时，输入的 `Title` 必须与添加时的标题**完全一致**（包括大小写和下划线等符号，例如 `Book_Title_1`），否则会触发 `ItemNotFoundError` 异常。
2. **异常提示**: 系统故意保留了一些非致命的异常提示（如查找失败时的 `Error: Not found` 或库存不足时的 `OutOfStockError`），这属于正常业务逻辑的一部分，旨在向用户反馈操作结果。
3. **数据重置**: 如果您希望清空所有数据重新开始，只需手动删除项目目录下的 `data/library_data.json` 文件，再次运行系统即可重新生成初始数据集。
